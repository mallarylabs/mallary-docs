Mallary API is the public API for multi-platform publishing, analytics, media upload, and webhooks.

  <ol>
    <li>Sign up at https://mallary.ai</li>

Authenticate every request with your Mallary API key:

```http
Authorization: Bearer <mallary_api_key>
```

The API key owner is authoritative. Do not use `user_id` to identify the caller.

## Base URL

```text
https://mallary.ai
```

## Core API Features

The current OpenAPI spec documents:

- presigned media uploads
- post creation and scheduling
- job status retrieval
- grouped post listing
- deletion of queued or scheduled posts
- analytics snapshots
- platform disconnects
- webhook listing, creation, and deletion

## Supported Platforms

The current public platform enum includes:

- `twitter`
- `x`
- `facebook`
- `instagram`
- `linkedin`
- `youtube`
- `tiktok`
- `pinterest`
- `reddit`
- `google_business`
- `snapchat`

## Posting Features

`POST /api/v1/post` supports:

- one or more target platforms
- media attachments
- `comments_under_post`
- optional scheduling via `scheduled_at`
- optional per-post AI auto-reply override
- per-platform `platform_options`
- optional `webhook_url`
- optional `Idempotency-Key`

## Rate Limits

Rate limits are enforced per user, per minute, by plan:

- plan `1`: `75 req/min`
- plan `2`: `150 req/min`
- plan `3`: `750 req/min`
- plan `4`: `1500 req/min`

## Plan-Gated Features

Some endpoints and features are plan-gated.

Examples called out in the spec include:

- posting limits
- analytics availability
- AI auto-reply availability

## Flow Example: video post to multiple platforms

<Steps>
<Step title="Get a presigned URL for your video">

```typescript post-a-video.ts
const uploadMetaRes = await fetch("https://mallary.ai/api/v1/upload", {
  method: "POST",
  headers: {
    Authorization: `Bearer ${API_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    filename: file.name, // launch.mp4
  }),
});

const uploadMeta = await uploadMetaRes.json();

// Example output:
// {
//   uploadUrl: "https://...",
//   mediaUrl: "https://files.mallary.ai/uploads/launch.mp4",
//   headers: { "Content-Type": "application/octet-stream" }
// }
```

</Step>
<Step title="Upload the video">

```typescript post-a-video.ts
await fetch(uploadMeta.uploadUrl, {
  method: "PUT",
  headers: uploadMeta.headers ?? {},
  body: file,
});

// Example output: HTTP 200/204 from uploadMeta.uploadUrl.
```

</Step>
<Step title="Create a post with the video attached">

```typescript post-a-video.ts
const postRes = await fetch("https://mallary.ai/api/v1/post", {
  method: "POST",
  headers: {
    Authorization: `Bearer ${API_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    message: "Check out our new product!",
    platforms: ["tiktok", "youtube", "facebook", "instagram"],
    media: [{ url: uploadMeta.mediaUrl }],
    comments_under_post: ["Find out more at https://example.com today"],
    auto_reply_enabled: true,
  }),
});

const post = await postRes.json();
const jobId = post.jobs[0].jobId;

// Example output:
// {
//   status: "queued",
//   batch_id: "a1b2c3d4",
//   jobs: [{ platform: "facebook", jobId: "123" }]
// }
```

</Step>
<Step title="Check the job status">

```typescript post-a-video.ts
const jobRes = await fetch(`https://mallary.ai/api/v1/jobs/${jobId}`, {
  headers: { Authorization: `Bearer ${API_KEY}` },
});

const { data } = await jobRes.json();

console.log(data.job);

// Example output:
// {
//   id: "123",
//   platform: "facebook",
//   status: "completed",
//   result: { postUrl: "https://..." }
// }
```

</Step>
</Steps>

## Core API Features

The current OpenAPI spec documents:

- presigned media uploads
- post creation and scheduling
- job status retrieval
- grouped post listing
- deletion of queued or scheduled posts
- analytics snapshots
- connected platform listing
- platform disconnects
- webhook listing, creation, and deletion

## Supported Platforms

The current public platform enum includes:

- `twitter`
- `x`
- `facebook`
- `instagram`
- `linkedin`
- `youtube`
- `tiktok`
- `pinterest`
- `reddit`
- `threads`
- `snapchat`

## Posting Features

`POST /api/v1/post` supports:

- one or more target platforms
- media attachments
- `comments_under_post`
- optional scheduling via `scheduled_at`
- timezone-aware scheduling via `scheduled_timezone` with IANA values like `America/New_York`
- optional per-post AI auto-reply override
- per-platform `platform_options`
- optional `webhook_url`
- optional `Idempotency-Key`

## Platform-Specific Media Rules

Mallary also enforces platform-specific media rules before jobs are queued.

Examples:

- Threads supports `text`, `image`, `video`, and `carousel` post types. Carousel posts support 2 to 20 supported media items.
- Scheduling supports absolute timestamps such as `2026-04-06T18:30:00Z`, or local wall-clock times such as `2026-04-06T14:30` when paired with `scheduled_timezone: "America/New_York"`.
- YouTube requires exactly one video.
- Pinterest requires exactly one image or GIF, or exactly one video, and also requires `boardId`.
- Reddit image posts require one image or GIF, while Reddit video upload is not supported by the current public API path.
- TikTok supports one video for video posts, or up to 35 images for photo posts.
- X allows up to 4 images or 1 video or 1 GIF.

For the full per-platform matrix and current runtime caveats, see:

[Platform-specific media rules](/api-reference/endpoint/create#platform-specific-media-rules)

## Rate Limits

Rate limits are enforced per user, per minute, by plan:

- plan `1`: `75 req/min`
- plan `2`: `150 req/min`
- plan `3`: `750 req/min`
- plan `4`: `1500 req/min`

## Plan-Gated Features

Some endpoints and features are plan-gated.

Examples called out in the spec include:

- posting limits
- analytics availability
- AI auto-reply availability

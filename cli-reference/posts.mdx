---

## Flag Mode

```bash
mallary posts create \
  --message "Hello from Mallary CLI" \
  --platform facebook \
  --platform linkedin \
  --media ./hero.png \
  --comment "Follow-up comment 1" \
  --comment "Follow-up comment 2" \
  --auto-reply-enabled \
  --scheduled-at 2026-03-30T15:00:00Z
```

## File Mode

```bash
mallary posts create --file ./post.json
```

Example:

```json
{
  "message": "Launch update",
  "platforms": ["facebook", "instagram", "linkedin"],
  "media": [{ "url": "./launch.png" }],
  "platform_options": {
    "instagram": {
      "post_type": "carousel"
    }
  }
}
```

## File Mode Rules

- `--file` is mutually exclusive with payload-building flags
- local `media[].url` file paths are uploaded to the Mallary CDN first
- existing remote media URLs must already be hosted on `https://files.mallary.ai/...`
- external remote media URLs are rejected
- platform-specific payloads are passed through to the API

## Platform-Specific Media Rules

Mallary CLI uses the same platform media validation as the Mallary API.

Important current rules:

- YouTube requires exactly one video.
- Instagram currently works best with one image or one video in the public publisher implementation.
- LinkedIn currently supports text-only posts or one image attachment only.
- TikTok currently requires one video file.
- Pinterest requires exactly one image or GIF, or exactly one video, plus `boardId`.
- Reddit image posts require one image or GIF, and Reddit video upload is not supported by the current public API path.
- Google Business currently requires one JPEG or PNG image.
- X allows up to 4 images, or 1 video, or 1 GIF.
- Snapchat requires partner/API access before posting can succeed, even though validation rules also exist for its media shapes.

For the full per-platform matrix and current runtime caveats, see:

[Platform-specific media rules](/api-reference/endpoint/create#platform-specific-media-rules)

## Comments Under Post

- use repeatable `--comment` flags in flag mode
- in file mode, send `comments_under_post` as an array
- the current limit is 3 items
- TikTok does not currently support `comments_under_post`

Example:

```json
{
  "message": "New launch today",
  "platforms": ["facebook"],
  "media": [{ "url": "./launch.png" }],
  "comments_under_post": [
    { "content": "What do you think?" },
    { "content": "Questions? Ask below." }
  ]
}
```

## AI Auto Reply

- available on Pro and Business plans only
- can be enabled account-wide in `mallary settings update`
- can be enabled per post with `--auto-reply-enabled`
- depends on saved account settings, not just the current payload

To enable it successfully, these settings must be populated:

- `business_name`
- `website_url`
- `business_description`
- `services`
- `contact_info`
